# Context

Aegis is built for the IBM Bob Hackathon.

## Core Positioning

Aegis is:

- Repository intelligence.
- Architecture understanding.
- Execution and feature flow tracing.
- Technical debt visibility.
- Modernization guidance.
- Developer onboarding acceleration.
- Impact analysis.

Aegis is not:

- A generic chatbot.
- A code generator wrapper.
- A replacement for IDE search.
- A static dashboard with no workflow.

## Design Principles

- Visual first.
- Developer first.
- Explainable analysis.
- Fast local iteration.
- Enterprise modernization relevance.
- AI-readable architecture and documentation.

## MVP Boundaries

The MVP focuses on local repository analysis, dependency graphing, trace approximation, debt scoring, and modernization suggestions.

It intentionally avoids authentication, cloud infrastructure, Kubernetes, and heavy platform concerns.
